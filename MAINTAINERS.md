# Maintainers
USTC Payment Portal

This document lists the maintainers responsible for the ongoing stewardship, quality, and operational health of the **USTC Payment Portal**.

The maintainer group is defined by the GitHub team that has Write/Admin access to this repository.

---

## 🔹 Maintainer Team

### **Primary Maintainers**

**Payment Portal Team**
GitHub Team:
`@ustaxcourt/payment-portal-team`

This team is responsible for:

- Reviewing and approving pull requests
- Overseeing architecture and long‑term technical direction
- Managing releases, deployment promotions, and rollback decisions
- Ensuring secure operation and compliance
- Maintaining high code quality, testing, and documentation standards
- Responding to incidents (in coordination with on‑call protocols)
- Coordinating with security reviewers for sensitive or high‑risk changes

> To view the current list of maintainers, visit the team page in the USTC GitHub organization’s settings.

---

## 🔹 Responsibilities & Expectations

Maintainers are expected to:

### 1. **Code Review & Quality**
- Provide timely code reviews
- Ensure consistency with project conventions
- Uphold security, stability, and maintainability
- Require tests and documentation updates for relevant changes

### 2. **Architecture & Direction**
- Ensure changes comply with architectural goals
- Approve or propose Architecture Decision Records (`/.adr-dir`)
- Collaborate on system evolution and improvements

### 3. **Operational Excellence**
- Validate staging before promotion
- Manage production deployments and monitor rollout
- Follow the incident playbook (`/docs/runbooks/incident-response.md`)
- Participate in post‑incident reviews when required

### 4. **Security**
- Guard against unsafe changes
- Enforce secure coding standards
- Coordinate vulnerability reporting per `/SECURITY.md`

### 5. **Documentation**
- Maintain clear and current docs
- Ensure new features or changes include updated documentation

## 🔹 How Maintainers Are Added or Removed

Changes to maintainer membership follow the governance model defined in:

**`/GOVERNANCE.md`**

Generally:

- New maintainers must be added to the
  `@ustaxcourt/payment-portal-team`
- Departing maintainers should be removed from the same team

All updates should be performed through standard organization‑level processes.
